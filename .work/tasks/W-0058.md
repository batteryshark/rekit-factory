## Goal
Make every rendered worker report a canonical outcome projection without allowing model-authored status prose to assert validation or acceptance.

## Requirements
- [x] Project a stable `report` entity parented to its canonical work item
- [x] Give a rendered report only the publication state `rendered`
- [x] Never infer validation, acceptance, completion, or disposition from `status_update`
- [x] Preserve exact full-rebuild and selective incremental parity for report add/update/removal
- [x] Return projection schema/vocabulary/semantic identity plus canonical report identity, parent, facets, and diagnostics from `/reports`
- [x] Render canonical facets in Mission Control and label escaped model prose as an unverified worker note

## Acceptance Criteria
- [x] A worker note claiming “validated and accepted” leaves both facets not-applicable
- [x] Removing or replacing the structured result removes the report in both folds
- [x] The reports API has no ambiguous `status` field
- [x] Mission Control distinguishes canonical facets from the worker note
