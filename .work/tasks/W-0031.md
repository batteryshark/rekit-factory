## Goal
Extend single remote invocations into provider-neutral, time-bounded analysis ranges for investigations that genuinely need multiple hosts, services, architectures, or interactive stations.

## Requirements
- [x] Define a versioned range specification with content-addressed template, nodes, platform/architecture/images, capabilities, services, topology, immutable inputs, network/egress policy, resource ceilings, lifetime, and reset/destroy behavior
- [x] Define a durable lease lifecycle such as requested, provisioning, ready, in-use, resetting, destroyed, expired, and failed, with exact controller ownership
- [ ] Keep hypervisor/cloud/range credentials in provider configuration; workers receive scoped capability/connection handles, never general infrastructure authority
- [ ] Default to isolated networks and no internet/LAN egress; derive all exceptions from W-0022 authorization
- [ ] Stage targets read-only, allocate per-lease scratch/output, collect verified artifacts, and reset or destroy all mutable state at lease end
- [ ] Support headless workers and audited optional attach actions without allowing attachment to bypass tool, scope, or evidence policy
- [ ] Provide adapter boundaries for local containers, native Windows workers, VM systems, and existing range products without choosing one as a Factory dependency
- [ ] Handle partial provisioning, controller crash, leaked lease, node failure, cancellation, expiration, and cleanup retries idempotently
- [ ] Record environment/image/tool versions and topology identity in proof bundles and benchmark comparison keys
- [ ] Prevent one campaign from discovering or reaching another campaign's lease

## Delivered Read-only Attachment Contract
- [x] Bind an observer request to the exact range, lease generation/revision, opaque node handle, scope digest, evidence-policy digest, actor, action, and expiry
- [x] Permit only server-declared `observe-console` and `capture-screen` actions; commands, keyboard input, provider endpoints, credentials, and host paths have no contract field
- [x] Emit deterministic allowed/denied audit records and fail closed for stale policy, wrong scope/node/lease, destroyed or expired leases, and cross-lease handles

This is the authorization boundary for an adapter-owned observer session, not proof that a real
provider enforces it. The broader attachment requirement remains unchecked until W-0077 persists
the audit before a real attach effect and proves the session cannot bypass scope, tool, or
evidence policy.

## Delivered Metadata and Mission Control Projection Contracts
- [x] Bind adapter, image, environment, tool, topology, template, spec, scope, and generation identities into one content-addressed execution record
- [x] Derive proof metadata and benchmark comparison keys from the same exact execution identity
- [x] Project bounded range health, node facts, machine failure state, and current-generation attachment audits without provider handles, credentials, endpoints, failure prose, or host paths
- [x] Validate, escape, and bound the Mission Control range renderer independently of provider-authored or hostile extra fields

These projections are ready for a real controller publisher, but do not satisfy the live
Mission Control acceptance criterion by themselves. W-0077 still owns real adapter health,
persisted audit-before-effect, and environmental proof.

## Acceptance Criteria
- [ ] A benign two-node offline fixture is provisioned, exercised, evidenced, reset, and destroyed through one transport-neutral contract
- [ ] Network tests prove no LAN/internet/cross-lease reachability under the default policy
- [ ] Controller restart during every lifecycle phase converges without duplicate ranges or abandoned mutable targets
- [ ] Expired/cancelled leases stop accepting work and eventually reach an auditable terminal cleanup state
- [x] The same investigation contract can target at least two adapter types in deterministic tests
- [ ] Mission Control can inspect range health and audited attachments without receiving provider credentials or private host paths

## Plan
Design the provider-neutral spec/lease state machine after W-0006 stabilizes single-worker transport. Implement a fake adapter and isolation tests, then one real local/offline two-node proof before considering external range products.

## Notes
A range is justified only when a test needs a topology or platform boundary. It should not replace lightweight local/remote invocations for ordinary static analysis.
