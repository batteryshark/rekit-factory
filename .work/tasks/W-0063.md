## Goal
Define a pure, fail-closed policy that turns verified canonical outcome transitions into stable redacted notification candidates without claiming delivery or outbox durability.

## Requirements
- [x] Require intact `factory-outcomes/v1` semantic identity and canonical bytes
- [x] Diff old-to-new canonical entities deterministically
- [x] Emit stable candidates for newly waiting decisions and reproduced/accepted findings
- [x] Suppress hydration, unchanged state, progress chatter, report prose, and self-resolved decisions
- [x] Fail closed for degraded projections, unknown facets, and unsafe identifiers
- [x] Include only fixed copy, safe opaque identities, and hashed transition dedupe keys
- [x] Keep persistence, delivery, retry, acknowledgement, batching, and channel behavior out of scope

## Acceptance Criteria
- [x] An unchanged projection or duplicate current-state replay emits no candidate
- [x] Re-evaluating the same threshold crossing produces the same dedupe key
- [x] A newly waiting decision emits exactly one redacted action-required candidate
- [x] Reproduced and accepted finding thresholds remain distinct
- [x] Raw target paths, prompts, evidence, and model-authored report prose cannot enter payloads
- [x] Degraded, tampered, unsupported, or unsafe inputs never produce an alert
