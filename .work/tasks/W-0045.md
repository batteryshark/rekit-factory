## Goal
Prove the W-0006 protocol against one real, disposable native Windows environment capable of a benign x64dbg-class workflow, without making a specific hypervisor a Factory dependency.

## Requirements
- [ ] Select and document one authorized adapter/environment whose image provenance, Windows/tool versions, license/use constraints, and reset mechanism are reproducible
- [ ] Implement Windows-safe state and path handling using handle/reparse-point/DACL semantics or keep the durable journal on a trusted POSIX controller; do not silently weaken W-0006's fail-closed guarantees
- [ ] Advertise exact capabilities and bind every invocation to the W-0022 scope revision, target hash, network/mount policy, lease identity, and W-0039 environment attestation
- [ ] Start from a known-clean image, stage a benign owned fixture read-only, run one bounded analyzer workflow, return verified logs/artifacts, and reset or destroy all mutable state
- [ ] Default to no LAN/internet egress and no host mounts; test every explicitly enabled exception and keep secrets environment-owned
- [ ] Exercise interruption, cancellation, controller/worker restart, dirty lease recovery, artifact mismatch, and failed reset without duplicate execution or false-ready state
- [ ] Record enough adapter-neutral evidence that a second implementation can conform without copying machine-specific controller logic

## Acceptance Criteria
- [ ] A clean Windows instance executes the benign fixture through the production W-0006 envelope and returns content-verified provenance-bound artifacts
- [ ] Reset/destroy returns the environment to the recorded baseline and a second run cannot observe the first run's scratch state
- [ ] Network tests prove the declared default isolation and any granted endpoint exception exactly
- [ ] Restart and failure injection converge to an auditable terminal or dirty state without replaying an invocation
- [ ] Native path aliases, junctions/reparse points, ACL drift, and staging escapes fail closed
- [ ] No claim depends on the discarded Parallels experiment or on a fake adapter being treated as Windows evidence

## Plan
Do not start merely because a VM is available. First select an authorized disposable adapter and threat model, then implement the smallest native boundary, run the benign lifecycle fixture, and publish a proof dossier whose exact environment and reset identity can gate later capstone work.
