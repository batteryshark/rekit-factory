## Goal
Let cron, CI, n8n-like workflow systems, and small deterministic services launch approved investigations and consume redacted outcomes without duplicating Factory orchestration or bypassing human/safety gates.

## Requirements
- [ ] Define authenticated, versioned command endpoints for launching from approved templates/scope envelopes, cancelling, requesting status, and acknowledging handoffs
- [ ] Require client idempotency keys so retries cannot create duplicate campaigns, runs, answers, or external effects
- [ ] Expose a durable cursor-based redacted event/outbox feed for consequential transitions and proof/report availability
- [ ] Keep Muster work, Factory campaign state, permission questions, authorization, model sessions, and evidence canonical inside their existing owners
- [ ] Do not allow external schedulers to supply provider secrets in job payloads or answer gated decisions unless a separately configured machine-policy explicitly authorizes that exact decision class
- [ ] Validate target references against configured roots/staging services and W-0022 scope; never accept arbitrary host paths from a remote client
- [ ] Add signed requests or mutually authenticated transport, replay protection, bounded payloads/rates, audit events, and credential rotation guidance before non-loopback deployment
- [ ] Support schedule metadata and origin attribution without building a second scheduler database into Factory
- [ ] Return stable run/campaign IDs and deep links; delivery/consumer outages cannot corrupt or roll back canonical state
- [ ] Document a minimal generic webhook/CLI example instead of coupling Factory to a particular automation vendor

## Acceptance Criteria
- [ ] Replaying the same launch request returns the existing campaign and does not duplicate work
- [ ] A scheduler cannot broaden scope, inject credentials, access private filesystem paths, or auto-approve a human gate
- [ ] Cursor resume delivers every consequential event once logically even when transport retries duplicate bytes
- [ ] An external consumer outage and later recovery does not block or lose the Factory run
- [ ] Authentication/replay/rate/payload failures are audited and fail closed
- [ ] A deterministic sample workflow launches an owned fixture, waits for a terminal/redacted event, and fetches its verified dossier

## Plan
Build idempotent loopback semantics and fake-consumer tests first. Reuse W-0030 outbox for outbound delivery. Add hardened non-loopback transport only with an explicit deployment threat model.

## Notes
The valuable n8n idea is deterministic integration around the reasoning system. n8n or another scheduler must never become a co-owner of work state, credentials, evidence, or approvals.
