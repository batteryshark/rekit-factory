## Goal
Persist the bounded canonical campaign health facts Mission Control needs without deriving policy state in the API or browser.

## Requirements
- [ ] Store content-bound coverage/outcome totals, novel-progress counts, retry/no-progress counters, phase, elapsed observation, and next-checkpoint expectation with each applied epoch recommendation
- [ ] Rebuild and validate every health rollup from the campaign event history; expose only bounded problem counts/codes when replay is degraded
- [ ] Preserve exact campaign/epoch/checkpoint/policy-input identity and reject cross-campaign or regressing facts
- [ ] Bound retained history and public projection size while keeping current and comparison values

## Acceptance Criteria
- [ ] Restart/crash fixtures reproduce the same rollup exactly once without rerunning work
- [ ] Coverage, novelty, retry/no-progress, elapsed, and next-checkpoint values cannot be forged by API/browser callers
- [ ] Large histories, corrupt replay, concurrent campaigns, and missing observations fail closed with bounded diagnostics
