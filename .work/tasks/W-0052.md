## Goal
Add the durable campaign/epoch/checkpoint state needed to resume exactly once after interruption without creating a second run, evidence, memory, or decision database.

## Requirements
- [x] Store canonical W-0051 identities and transition events transactionally beside existing Factory/Muster authority
- [x] Make campaign, epoch, lease, checkpoint, and operator-decision operation IDs idempotent; conflicting reuse fails closed
- [x] Commit budget consumption, work publication, checkpoint references, and terminal transition in an order that cannot expose partial authority
- [x] Recover running/leased work conservatively after controller or host restart and never infer completion from a worker disappearance
- [x] Rebuild the campaign projection from canonical state and detect gaps, duplicates, dangling references, and impossible transitions without rewriting history
- [x] Add deterministic crash injection at every write boundary and concurrent-campaign isolation tests

## Acceptance Criteria
- [x] Restart at every epoch transition resumes or blocks exactly once with no duplicated work or spend
- [x] An interrupted checkpoint is either wholly visible or absent; prior committed state remains byte-identical
- [x] Concurrent campaigns cannot read or mutate one another's checkpoints, budgets, approvals, work, or evidence
- [x] Full rebuild matches the live projection and surfaces corrupt/stale history as degraded

## Notes
Reuse existing ledgers and project-memory references. Do not create a transcript store or mutable shadow source of truth.
