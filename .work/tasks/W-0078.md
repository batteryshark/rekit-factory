## Goal
Connect the loopback automation gateway to the canonical investigation controller and one environment-owned approved fixture catalog without creating another work ledger or approval authority.

## Requirements
- [x] Implement an `AutomationOwner` adapter over `InvestigationController` with restart-stable operation identity
- [x] Configure an environment-owned catalog that resolves opaque template, target, and exact W-0022 scope-envelope references
- [x] Make launch, active-run cancellation, status, and handoff acknowledgement delegate to their canonical owners idempotently
- [x] Project only committed consequential notification and verified dossier events into the automation cursor feed
- [x] Return stable run/project IDs, include a campaign ID only when a canonical campaign exists, and provide exact Mission Control deep links without exposing roots, paths, secrets, scopes, or gate answers
- [x] Reconcile interruption before and after every owner call without duplicate work or false success

## Acceptance Criteria
- [x] A deterministic owned fixture launches through the gateway, reaches a terminal canonical outcome, and exposes its verified dossier through cursor resume
- [x] Exact launch replay before and after restart returns the same run and creates no duplicate Muster work
- [x] Cancellation and acknowledgement replays preserve canonical state and cannot answer an unrelated or human-only gate
- [x] Missing/stale catalog entries and hostile client fields fail closed with redacted durable audits
- [x] Consumer outage and recovery do not block, lose, or roll back investigation progress
