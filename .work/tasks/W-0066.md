## Goal
Make terminal run state, the terminal event, canonical investigation export, report registry row, and `run.json` mirror converge after interruption or derived-material deletion.

## Requirements
- [ ] Enumerate and deterministically inject failure after terminal run commit, terminal event, report atomic replace, report-row insert, and run-status mirror replace
- [ ] Add one idempotent `reconcile_terminal_publications` boundary that derives every repair from canonical run/event/outcome state
- [ ] Produce exactly one reason-matching terminal event and one report registry row while preserving a verified canonical export
- [ ] Rebuild deleted or corrupt derived report/coverage/summary/status material without changing canonical outcome meaning or dossier visibility

## Acceptance Criteria
- [ ] Restart after every injected boundary converges without duplicate terminal events, report rows, or false nonterminal mirrors
- [ ] Deleting `investigation.json`, its report row, coverage/summary mirrors, or stale `run.json.status` and reconciling preserves the canonical outcome semantic SHA and `dossier_list`
- [ ] A conflicting noncanonical export or terminal reason fails closed with a bounded diagnostic
