## Goal
Establish the provider-neutral, content-addressed v1 contract and deterministic fake lifecycle for bounded multi-node analysis ranges before choosing or invoking any real hypervisor, cloud, native-Windows, or commercial range adapter.

## Requirements
- [x] Define strict versioned contracts for range template/spec identity, nodes, platform/architecture/image, capabilities/services, topology, immutable inputs, scope revision, network/egress, resource ceilings, lifetime, reset/destroy policy, and provider handle references
- [x] Define exact requested/provisioning/ready/in-use/resetting/destroyed/expired/failed transitions with owner, terminality, allowed predecessors, and reasoned failure fields
- [x] Keep provider credentials and private host paths outside every contract; handles are opaque, bounded, and never grant general infrastructure authority
- [x] Implement a deterministic in-memory fake adapter for a benign two-node offline fixture with staged read-only inputs, per-lease scratch/output, evidence metadata, reset, destroy, expiry, and injected failures
- [x] Make exact retries idempotent and conflicting range/operation-ID reuse fail closed across a serializable fake checkpoint/restart
- [x] Enforce isolated-network/no-egress defaults and W-0022 scope-derived exceptions without claiming real packet-level isolation

## Acceptance Criteria
- [x] Canonical encoding and content identities are stable under mapping/set-like reorder and change on every security-relevant field
- [x] The two-node fake provisions, becomes ready, accepts bounded work, records verified-output metadata, resets, and destroys deterministically
- [x] Restart and failure injection at every lifecycle transition converge without duplicate leases or false-ready state
- [x] Expired/cancelled/destroyed leases reject new work and cleanup is idempotent
- [x] Cross-lease node/handle/input access and undeclared network/credential/mount intent fail closed
- [x] Documentation explicitly says fake lifecycle conformance is not real Windows, hypervisor, network-isolation, or disposal proof

## Plan
Land dependency-free contracts and transition rules first, then the serializable fake adapter and adversarial lifecycle tests. Production controller wiring and real adapter proofs remain separate W-0031/W-0045 slices.

This task is intentionally safe to start after W-0006. It does not start W-0045, provision infrastructure, acquire a target, or satisfy the deferred Frenzy Express capstone.
