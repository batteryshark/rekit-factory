## Goal
Produce an evidence-backed clean-room specification of the exact `xio.dll` interface used by the authorized Frenzy Express build, sufficient for an independent minimal keyboard implementation.

## Requirements
- [ ] Inventory exports, ordinals, imports, calling conventions, parameter/return layouts, global state, object lifetime, and error behavior
- [ ] Trace initialization, configuration, device discovery, polling/event delivery, focus transitions, key mapping, timing, shutdown, and relevant threading assumptions
- [ ] Distinguish observed behavior, statically inferred behavior, hypotheses, and independently reproduced facts
- [ ] Record exact target/module hashes and cite bounded disassembly, traces, call-site observations, and experiments without copying proprietary implementation code
- [ ] Define a versioned behavioral contract and conformance vectors for the smallest keyboard-only subset
- [ ] Document unknowns and discriminating tests instead of inventing semantics for unobserved paths

## Acceptance Criteria
- [ ] Every claimed entry point and structure field links to reproducible evidence or is explicitly marked unresolved
- [ ] A call-sequence/state-machine model explains observed startup, polling, focus loss/recovery, and shutdown behavior
- [ ] Test vectors cover ordinary keys, simultaneous keys, repeats/transitions, unmapped inputs, and invalid call order where observed
- [ ] An independent worker can begin W-0038 using only the public contract, fixtures, and evidence references rather than proprietary source or copied code
- [ ] The documentation clearly states build/hash applicability and does not generalize beyond tested evidence

## Plan
Start from imports and call sites, form a small set of interface hypotheses, and use controlled observation to eliminate ambiguity. Publish the contract and conformance vectors before any compatibility implementation begins.

This clean-room research lane can run in parallel with W-0036 after W-0035. Documentation is a first-class deliverable, not an incidental byproduct of emulator coding.
