## Goal
Demonstrate that one v1 investigation contract and durable lifecycle can target two genuinely distinct deterministic adapter types without coupling checkpoint recovery to one handle namespace or evidence encoding.

## Requirements
- [x] Add a second inert adapter type with a distinct opaque provider-handle namespace
- [x] Give the second adapter a deterministic evidence payload shape distinct from the original fake
- [x] Reuse the strict v1 lifecycle and checkpoint validation boundary across both adapters
- [x] Keep both adapters free of real provisioning, command execution, network, credential, and host-path effects

## Acceptance Criteria
- [x] The benign two-node contract provisions and executes through both adapters
- [x] Exact operation retry survives checkpoint/restart for both adapters
- [x] Reset clears lease-local outputs, preserves evidence, increments generation, and invalidates old handles for both adapters
- [x] Destroy remains idempotent after restart for both adapters
- [x] Injected work and destroy failures replay exactly after restart and recover through reset or cleanup for both adapters
- [x] Focused range tests pass

## Plan
Introduce explicit deterministic handle/evidence hooks in the existing conformance engine, implement a manifest-shaped adapter through those hooks, and parameterize lifecycle and failure-recovery tests over both adapter classes.

## Notes
This slice proves adapter boundary and deterministic state-machine parity only. It makes no claim about actual VM/container/native execution, network isolation, secure reset, or cross-tenant separation.
