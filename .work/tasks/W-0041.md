## Goal
Build one byte-deterministic, offline public package from an explicitly owned development fixture while keeping every private scoring and source artifact outside the archive by construction.

## Requirements
- [ ] Package only canonical sanitized public task metadata, public questions, and exact content-bound target bytes
- [ ] Derive the v1 public-package reference and bind every manifest path, size, media type, and SHA-256 digest
- [ ] Keep source, ground truth, private tests, prior dossiers, private paths, and held-out/evaluation identifiers outside the archive allowlist
- [ ] Normalize entry order, modes, ownership, timestamps, compression, and manifest serialization
- [ ] Reject symlinks, absolute or traversing paths, undeclared files, target digest/size drift, and non-development tasks
- [ ] Add one owned offline fixture with recorded ownership, license, authorization, build/reset metadata, and positive plus reproducible-negative questions

## Acceptance Criteria
- [ ] Unchanged canonical input produces byte-identical archive bytes and references across repeated offline builds
- [ ] Changing public metadata or target bytes changes the package digest
- [ ] An exact archive allowlist and sentinel scan prove private names and values are absent from names and bytes
- [ ] Every extracted target byte, manifest size/hash, and safe path verifies against the package reference
- [ ] The fixture round-trips through the v1 contracts and never compiles or uses the network during packaging tests

## Plan
Keep the packager, its focused tests, and one development fixture in an isolated ReverseBench worktree. A later W-0040 integration pass will add shared exports and cross-lane scoring/storage coverage.

## Notes
This proves deterministic exclusion from the produced development archive, not held-out secrecy, worker filesystem/retrieval isolation, reset safety, target execution, or cross-toolchain reproducibility. It does not loosen W-0028.
