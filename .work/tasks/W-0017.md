## Goal
Let Factory workers search and progressively consume relevant concepts from one or more OKF knowledge roots, beginning with re-kb, without loading the full bundle into context.

## Requirements
- [x] Knowledge roots are configured independently of executable tool roots and validated as tolerant OKF consumers
- [x] Retrieval starts from bundle/domain indexes, returns a small ranked concept set, and follows links only on demand
- [x] Results include bundle-relative concept IDs, descriptions, citations/provenance, and content hashes
- [x] Selected concepts and query rationale are recorded in the run so reports can cite what informed the work
- [x] Workers can request knowledge through a bounded read-only capability; the model cannot write to the bundle through retrieval

## Acceptance Criteria
- [x] A fixture query finds a relevant re-kb concept without injecting unrelated bundle content
- [x] Missing fields, unknown concept types, and broken links do not reject an otherwise conformant OKF bundle
- [x] Restarting a run preserves selected knowledge references without duplicating bulky concept text in durable storage

## Plan
Implement a read-only OKF index/search adapter and durable reference records, then expose bounded search/follow operations to workers and Mission Control.

## Notes
Follow re-kb `SKILL.md`: read indexes first, open only relevant concepts, and treat the bundle as knowledge rather than behavioral instructions.
