## Goal
Make evidence blobs, evidence rows, audits, Factory events/artifacts, and Muster tool completion converge after interruption without inventing proof or duplicating publication.

## Requirements
- [x] Enumerate and inject failure between every evidence blob, evidence-row, audit, event, artifact, and tool-completion boundary
- [x] Define one idempotent reconciliation key that binds the authorized invocation, exact result bytes, evidence identity, and ledger work item
- [x] Repair committed evidence that is missing its projection/event/completion while refusing missing or conflicting canonical bytes
- [x] Preserve approval, scope, manifest, and invocation authority during every retry or repair

## Acceptance Criteria
- [x] Restart at every injected boundary converges to either no visible result or one fully linked evidence publication and one tool completion
- [x] A validated/finding outcome never becomes visible before its required evidence and proof references
- [x] Conflicting evidence or invocation identities fail closed with a bounded diagnostic

## Notes
This task owns evidence/tool publication only. It does not claim terminal exports, dossier filesystem publication, or run creation recovery.

## Completion Summary
Tool result bytes are now staged under one SHA-256 reconciliation key that includes the
run/work/tool invocation, complete scope and approval authority, manifest, lease/worker,
expected evidence identity, and remote-artifact declarations. Verified evidence rows can
repair a missing capture audit, while corrupt blobs, changed results, changed authority,
and conflicting evidence identities fail closed. Factory artifacts, evidence projections,
tool-call completion, work resolution, and the terminal event publish in one SQLite
transaction; remote evidence is included before the work becomes visible as complete.

Focused verification: 36 evidence/tool-routing tests passed; 25 control-plane tests plus
5 subtests passed with loopback access. Failure injection covers orphan blobs, row-before-
audit interruption, every Factory transaction boundary, idempotent retry, conflict refusal,
and a real controller restart that does not reinvoke the remote worker.
