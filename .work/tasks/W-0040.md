## Goal
Establish the dependency-safe ReverseBench foundation that can be built before real disposable evaluation workers exist: versioned contracts, sealed development packages, deterministic typed scoring, and append-only trial records.

## Requirements
- [ ] Define validated canonical JSON contracts for task ownership/license/authorization, development/evaluation split, target/build digests, scope actions/network, reset policy, domain/difficulty, questions, and scoring version
- [ ] Keep private ground truth structurally separate from public worker packages; packages contain only allowlisted content-addressed target bytes, sanitized questions, public metadata, and digests
- [ ] Represent submissions as typed observed/absent/unknown answers with confidence and exact W-0025 dossier/manifest evidence citations
- [ ] Score exact positive, negative, forbidden/hallucinated, and unsupported answers deterministically without a model grading free-form prose
- [ ] Make model, harness, strategy, tool profile, environment/image, and policy versions part of a canonical comparison key
- [ ] Store immutable, append-only trial records; exact retries are idempotent and conflicting trial-ID reuse fails closed
- [ ] Define a canonical Factory request/receipt boundary without importing Factory internals or reading its remote journal
- [ ] Provide one owned, offline, development-only fixture that exercises packaging and scoring without claiming held-out security

## Acceptance Criteria
- [ ] Missing ownership, license, authorization, target/build digest, reset policy, or scoring version is rejected
- [ ] A leakage test proves the public package contains no private paths, source, answer values, private tests, prior dossiers, or stable held-out identifiers
- [ ] A correct positive scores, an invented behavior is penalized, and a reproducible absence receives credit
- [ ] Every comparison-key component changes the key while trial timestamps/results do not
- [ ] Repeated trials remain distinct across restart and interrupted writes cannot replace a prior record
- [ ] Packaging and scoring are deterministic, offline, and byte-identical for unchanged canonical inputs
- [ ] Machine-readable contracts clearly distinguish what this development slice proves from W-0028's still-deferred held-out isolation and three-fixture acceptance

## Plan
Land the shared contract and Factory-boundary package first, then fan packaging, scoring, and append-only records into independent worktrees. Integrate one owned development fixture last and keep W-0028 open for real isolated trials, variance, release gates, and held-out leakage proof.

## Notes

Durable invocation journaling is not environmental isolation. No public challenge enrollment, third-party target, historical CTF import, live submission, or worker access to private ground truth is in scope.
