## Goal
Qualify one real, local, offline, resettable adapter against the existing v1 range contracts without generalizing beyond the exact tested provider, image, and policy.

## Requirements
- [ ] Select an available local container, VM, or equivalent provider and pin its adapter, image, environment, and tool identities
- [ ] Keep provider credentials in external configuration and give workers only lease-scoped connection/capability handles
- [ ] Provision the benign two-node fixture with read-only inputs, distinct bounded scratch/output, isolated networking, and no default LAN/internet egress
- [ ] Derive every egress or attach exception from exact W-0022 authorization and persist the read-only attachment audit before any provider effect
- [ ] Collect content-verified outputs and bind environment, image, tool, topology, scope, and reset identity into proof/comparison metadata
- [ ] Supervise partial provisioning, crash, node failure, cancellation, expiration, reset, destroy, and cleanup retry without duplicate provider ranges
- [ ] Enforce campaign ownership at discovery and every lease/node/attachment operation
- [ ] Project bounded range health and attachment audit state to Mission Control without provider credentials or private host paths

## Acceptance Criteria
- [ ] One benign two-node offline lease is provisioned, exercised, evidenced, reset, and destroyed through the transport-neutral contract
- [ ] Probes prove no LAN, internet, or cross-lease/campaign reachability under the default policy
- [ ] Restart injection at every lifecycle phase converges without duplicate ranges or abandoned mutable state
- [ ] Expired and cancelled leases reject work and eventually reach an auditable cleanup terminal state
- [ ] A permitted read-only observer attach is audited and cannot provide input or bypass scope/tool/evidence policy
- [ ] Mission Control renders exact bounded health and attachment audits with no provider secret or host-path disclosure
