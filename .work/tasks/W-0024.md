## Goal
Prevent plausible model narratives from becoming accepted findings by requiring explicit evidence, a reproducible demonstration, and policy-selected independent validation.

## Requirements
- [x] Define a finding lifecycle that distinguishes lead, candidate, demonstrated, reproduction-pending, reproduced, operator-accepted, rejected, withdrawn, and inconclusive
- [x] Require candidate findings to cite the hypothesis, target scope, observation evidence, affected artifact/component, impact claim, assumptions, and known uncertainty
- [x] Define proof policy by finding type and consequence; informational observations may need less than vulnerability, exploitability, or destructive-impact claims
- [x] Generate validation work that is independent from the originating assertion: a different worker/session and a clean environment; optionally a different model/profile for higher assurance
- [x] Ensure validators receive the proof recipe and material artifacts, not the originator's persuasive narrative or unsupported confidence
- [x] Record exact reproduction attempts, negative results, environmental differences, flakiness, and contradictions
- [x] Require at least one successful clean reproduction before a finding can be called reproduced; operator acceptance remains separate
- [x] Permit explicit waivers only as operator decisions that preserve the unmet criterion and never relabel unverified work as reproduced
- [x] Link severity/confidence changes to evidence and enforce monotonic provenance even when conclusions are downgraded or withdrawn
- [x] Keep run completion, coverage completion, finding reproduction, and operator acceptance as separate states

## Acceptance Criteria
- [x] A persuasive but unsupported model finding remains a candidate and is excluded from validated reports
- [x] A deliberately flaky/false-positive fixture fails independent reproduction and records the contradictory evidence
- [x] A valid fixture reproduces from a clean environment using only the recorded recipe and staged inputs
- [x] Reusing the originator's worker/session cannot satisfy an independence policy that requires separation
- [x] Restarting during validation does not duplicate reproduction attempts or lose their terminal results
- [x] Withdrawing or downgrading a finding preserves all prior evidence and decisions without leaving stale validated projections

## Plan
Implement finding/transition/proof-policy records and pure validation first. Add validator work generation and clean-environment execution second. Add operator acceptance and report filtering only after restart-safe reproduction works.

## Notes
The validator checks observable claims, not whether the original worker's reasoning sounded good. For reverse engineering, independent reproduction can mean a second execution path, a different static/dynamic technique, or a clean rebuilt environment depending on the claim.
