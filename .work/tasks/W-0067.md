## Goal
Complete W-0056's SQL transaction proof by covering dossier staging, verification, content-addressed rename, cleanup, and the final-directory-to-SQL visibility gap.

## Requirements
- [x] Inject deterministic failure after every staged included artifact, manifest, proof, Markdown/HTML report, ZIP, staged verification, final rename, and before SQL publication
- [x] Keep every dossier hidden from SQL/API until the complete final directory verifies
- [x] Reconcile orphan staging and valid-but-unpublished final directories safely after restart
- [x] Reuse byte-identical valid final content and publish exactly five linked artifacts plus one event once
- [x] Fail closed on a conflicting content-addressed final directory without overwriting evidence

## Acceptance Criteria
- [x] Restart/retry at every filesystem boundary converges to absent or fully verified published state, never partial visibility
- [x] Stale staging is removed or ignored deterministically and cannot shadow a valid final dossier
- [x] The existing W-0056 SQLite rollback guarantees remain intact across the expanded matrix

## Notes
This is the filesystem complement to W-0056's middle-artifact/final-event SQL injection tests. It does not claim terminal export, evidence store, or run-creation reconciliation.
