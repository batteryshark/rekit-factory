## Goal
Produce a small, verifiable proof bundle and human-readable dossier for each consequential finding so an operator can reproduce, audit, export, and compare the work without reconstructing it from chat logs.

## Requirements
- [x] Define a versioned proof manifest containing target/input hashes, scope revision, run/work/hypothesis/finding IDs, worker/environment identities, tool and capability versions, invocation arguments, network policy, evidence artifacts, expected/observed outcomes, reproduction steps, and validation verdicts
- [x] Use content-addressed relative artifact references with byte size and media type; reject absolute/traversing paths and verify hashes on read/export
- [x] Represent ordered steps as structured argv/actions plus environment facts, not an opaque shell transcript
- [x] Preserve platform/architecture/tool-version prerequisites and distinguish required inputs from optional visual aids
- [x] Generate deterministic JSON plus Markdown and HTML projections from the canonical manifest and finding state
- [x] Include observations, interpretations, limitations, contradictions, failed attempts relevant to reproduction, and operator decisions
- [x] Exclude credentials, raw private target bytes unless explicitly exportable, provider chain-of-thought, unrelated screenshots, and unredacted noisy logs
- [x] Make dossier/report visibility a projection from durable artifacts; never require a separate mutable indexing/sync job
- [x] Provide an offline verifier that checks schema, references, hashes, scope/export policy, and reproduction prerequisites without executing the proof
- [x] Support portable evidence references for knowledge candidates while honoring retention and private-evidence policy

## Acceptance Criteria
- [x] A reproduced fixture yields byte-identical manifests and rendered reports when canonical state is unchanged
- [x] Modifying any included artifact or manifest field causes verifier failure and cannot leave a stale valid badge
- [x] A clean worker can follow the dossier to reproduce the stated observation with no access to the original chat
- [x] A private/non-exportable artifact remains an opaque citation and is not copied into an export
- [x] A denied, inconclusive, withdrawn, or unreproduced finding is visibly labelled and cannot render as validated
- [x] Mission Control can open the dossier directly from the run ledger immediately after its artifact event is committed

## Plan
Land manifest schema/verifier, then deterministic renderers, then clean-worker replay assistance and Mission Control/report integration. Reuse remote invocation and artifact envelopes instead of creating another provenance format.

## Notes
The dossier is the useful part of the video's writeup surface; automatic blog-style decoration is not. Correct status, concise evidence, and reproducibility outrank screenshots and presentation polish.
