## Goal
Demonstrate with one real, resettable adapter that an evaluation worker can access only a sealed ReverseBench public package and approved output channel—not source, ground truth, private tests, prior dossiers, retrieval leaks, sibling trials, or host state.

## Requirements
- [ ] Select one authorized adapter/environment based on isolation and reset evidence rather than platform preference; a local container, VM, native service boundary, or range adapter may qualify
- [ ] Bind the exact image/environment, adapter, W-0006 worker, W-0022 scope, W-0025 evidence policy, W-0040 package, network/mount policy, and reset identity into the trial comparison/proof record
- [ ] Mount or stage only the sealed content-addressed public package read-only plus a fresh bounded scratch/output channel; expose no repository, private fixture tree, host home, credentials, sockets, or unrelated artifact/log stores
- [ ] Disable LAN/internet/default credentials and prove every allowed endpoint or attach capability from exact scope rather than adapter defaults
- [ ] Run adversarial probes for paths, environment, process metadata, retrieval/search, logs, caches, artifacts, sibling leases, and post-reset residue without teaching the evaluated worker hidden answer values
- [ ] Reset or destroy mutable state after success, failure, timeout, cancellation, and controller/worker restart; preserve only verified returned evidence outside the worker
- [ ] Produce a reproducible isolation dossier and explicit residual-risk statement; do not generalize beyond the tested adapter/image/policy versions

## Acceptance Criteria
- [ ] An owned canary fixture proves private source/truth/test/dossier sentinels are unreachable through every worker-visible channel while the public target remains usable
- [ ] Parallel trials cannot discover each other's package, scratch state, outputs, handles, processes, or network namespace
- [ ] A second trial after reset cannot recover the first trial's files, caches, environment, process state, or credentials
- [ ] Controller/worker interruption at every lifecycle boundary converges to clean reset/destroy or a durable dirty/blocked state; never false-ready
- [ ] Returned artifacts and logs are content-verified, scope-bound, redacted, and contain no hidden canary values
- [ ] The proof is strong enough to unblock held-out W-0028 work but makes no native-Windows claim unless W-0045 is independently satisfied

## Plan
Do not begin until an adapter can be exercised for real. Start with the W-0040 owned package and non-answer canaries, lock down every ingress/egress, inject failures and cross-trial probes, reset, then publish the exact environment-bound proof before enabling held-out fixtures.
