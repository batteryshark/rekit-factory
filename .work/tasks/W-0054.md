## Goal
Compose finite epochs, durable state, progress policy, scope enforcement, workers, and operator decisions into a restart-safe campaign controller that can never become an infinite prompt loop.

## Requirements
- [x] Launch only a bounded W-0051 epoch whose reserved work and spend fit remaining hard ceilings
- [x] Support serial and parallel workstreams while preserving global leasing, dedupe, scope, and budget invariants
- [x] Checkpoint at every epoch boundary, apply W-0053 policy exactly once, and publish the next epoch or terminal outcome transactionally through W-0052
- [x] Pause/stop/resume safely; require operator decisions for scope, hard-ceiling, forbidden-capability, or risk-threshold changes
- [x] Handle model, worker, controller, host, tool, and environment interruption without duplicate work or false success
- [x] Produce bounded evidence-referenced handoff and terminal summaries rather than raw transcript dumps

## Acceptance Criteria
- [x] A deterministic overnight simulation survives repeated restarts and completes every transition exactly once
- [x] Productive, no-progress, blocked, policy-stop, operator-stop, exhausted, and infrastructure-failure fixtures end with distinct reasoned outcomes
- [x] Parallel campaign fixtures cannot borrow scope, budget, approvals, workers, evidence, or storage
- [x] No test can spend or schedule beyond an epoch or cumulative ceiling even under concurrency and crash injection
