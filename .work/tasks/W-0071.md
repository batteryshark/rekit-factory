## Goal
Let an operator inspect and decide an exact server-published campaign change request without allowing the browser to construct or broaden authority.

## Requirements
- [ ] Persist and expose bounded pending `CampaignChangeRequest` documents and their exact request identities
- [ ] Add revision-bound idempotent approve/reject API operations that consume only a server-published request identity
- [ ] Apply an approved change as new content-bound campaign authority and never mutate scope, budgets, criteria, policy, or component versions from browser fields
- [ ] Render an e7 approval diff for scope, ceilings, completion, risk/capabilities, and versions with keyboard/reconnect/privacy safeguards

## Acceptance Criteria
- [ ] Forged, stale, changed, cross-campaign, and replayed request identities fail closed
- [ ] Repeated exact decisions converge once across reconnect and controller restart
- [ ] The browser cannot submit replacement contract content or bypass forbidden-capability/risk gates
