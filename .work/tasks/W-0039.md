## Goal
Make every Rekit capability declare the semantic authorities it may exercise so Factory can enforce W-0022 scope from a versioned catalog contract instead of inferring consequences from network and input-execution flags or conflating authorization with safety tier.

## Requirements
- [x] Extend the compatible Rekit manifest schema with explicit action authorities such as read, execute, modify, network, credential use, account creation, challenge enrollment/submission, persistence, destructive change, and third-party messaging
- [x] Keep safety tier, isolation needs, and operator tool consent separate from target/action authorization metadata
- [x] Define conservative compatibility behavior for legacy manifests; ambiguity must narrow availability or require review rather than grant authority
- [x] Validate declared authorities against network mode, target mutation, credential/account inputs, dispatcher behavior, and any prohibited combinations
- [x] Carry the declarations through public/private federated roots without leaking private root paths or credentials
- [x] Make Factory derive creation and dispatch scope checks from the declaration plus exact runtime intent; model arguments cannot weaken the manifest floor
- [x] Version and hash the effective manifest so approvals and proof records can cite the exact authority contract used
- [x] Migrate public Rekit and compatible `rekit-private` manifests with reviewable, least-authority declarations

## Acceptance Criteria
- [x] A read-only offline capability works under the minimum exact-target grant
- [x] A mutating, networked, credential-using, account-creating, or submitting fixture is denied unless every separately declared authority and exact runtime intent is approved
- [x] A low safety tier cannot bypass missing semantic authority, and operator consent cannot widen target scope
- [x] Unknown, malformed, contradictory, or omitted high-impact declarations fail closed with an auditable reason
- [x] Federated public/private capability projections expose safe authority names and content identity without source paths or secret values
- [x] Existing manifest compatibility is deterministic and covered by migration tests rather than an undocumented inference

## Plan
Version the shared registry contract and validator first, migrate representative public/private tools, then replace Factory's compatibility inference with declaration-driven scope requests while retaining a fail-closed legacy bridge.

This task owns capability semantics, not run authorization. W-0022 remains the policy and decision matrix; W-0039 tells it what a tool may actually do.
