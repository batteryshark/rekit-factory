## Goal
Bind the existing Parallels candidate to strict, content-addressed lifecycle identities and fixed `prlctl` argv without adding any command runner or provider effect.

## Requirements
- [x] Bind adapter version, binary/config/base-image digests, source VM, and source snapshot into one immutable adapter identity
- [x] Bind range, node, generation, spec, scope, and adapter identity into a deterministic logical VM identity and provider-safe clone name
- [x] Construct only exact linked-clone, start, ACPI-stop, snapshot-create, snapshot-switch, and delete argv from typed identities
- [x] Require an observed provider VM UUID after clone and an observed snapshot UUID before snapshot switch
- [x] Reject unknown fields, boolean schema versions, unhashable operation kinds, arbitrary argv, shell text, provider names, and cross-adapter identity reuse
- [x] Keep subprocess execution, output parsing, retries, provider state, credentials, and environmental assertions out of the module

## Acceptance Criteria
- [x] Equivalent typed inputs produce byte-stable command-plan identity and exact argv
- [x] Node, generation, scope, adapter binary, or provider identity changes split the command-plan boundary
- [x] Deserialization re-derives argv and rejects any extra flag or changed command
- [x] Tests prove the module exposes no subprocess-backed execution surface
- [x] Installed CLI help evidence supports the encoded fixed argument ordering

Read-only help from installed `prlctl version 26.4.0 (57513)` confirms `clone <ID|NAME> --name <new_name> --linked --id <snapshot>`, `snapshot-switch <ID|NAME> --id <snapshot> --skip-resume`, and the encoded start, ACPI stop, snapshot, and delete action shapes. The implementation contains no subprocess import or runner, accepts no arbitrary executable or argv, requires provider UUIDs after clone, and cannot claim that a command ran. Focused Parallels/range contract tests pass.
