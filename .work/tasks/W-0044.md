## Goal
Make a selected model profile's concurrency ceiling authoritative for every Factory launch and restart path, including direct API callers that bypass Mission Control's client-side validation.

## Requirements
- [x] Resolve the selected profile and reject an over-ceiling run before creating directories, run metadata, ledger rows, events, or supervisor work
- [x] Enforce the same policy for CLI, Mission Control, and crafted API requests without maintaining separate ceiling rules
- [x] Recheck a persisted run plan against the selected runtime profile before resumed work can fan out
- [x] Preserve exact-at-limit launches and existing behavior for profiles whose ceiling permits the requested plan
- [x] Return a bounded, actionable error that identifies the profile and permitted concurrency without exposing credentials or private configuration

## Acceptance Criteria
- [x] A crafted API request above the selected profile limit is rejected and leaves no run behind
- [x] A request exactly at the limit is accepted
- [x] Selecting a different named profile applies that profile's own limit
- [x] A persisted plan cannot resume above a newly stricter runtime profile ceiling
- [x] Existing CLI and controller tests pass with one shared policy invariant

## Plan
Add one controller-owned invariant over the selected backend profile, apply it before durable creation and before execution/resume, and cover both direct controller and HTTP entry points in an isolated worktree.

## Notes
This task governs worker fan-out concurrency only. Model-call retry policy, campaign budgets, persisted named safety policies, and provider rate-limit adaptation remain separate concerns.
