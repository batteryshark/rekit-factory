## Goal
Add modern aspect-ratio support to the authorized Frenzy Express build while preserving gameplay geometry, UI usability, and a reversible original experience.

## Requirements
- [ ] Characterize resolution discovery, viewport/projection math, UI layout, clipping, culling, video, cursor/input mapping, and configuration persistence with cited evidence
- [ ] Prefer a reproducible external patcher, loader, configuration adapter, or byte-delta recipe over committing modified proprietary binaries
- [ ] Support at least one 16:9 and one 16:10 mode; treat ultrawide as a measured extension rather than stretching by default
- [ ] Preserve intended field of view and gameplay timing unless an explicitly documented compatibility compromise is required
- [ ] Keep the change independently reversible and reject unsupported target hashes
- [ ] Produce before/after measurements and only the minimum meaningful visual evidence permitted by W-0023

## Acceptance Criteria
- [ ] A clean authorized baseline can apply and remove the widescreen change deterministically
- [ ] Supported resolutions render without simple stretch, critical UI cutoff, incorrect pointer mapping, or newly exposed invalid geometry in the test scenes
- [ ] An unknown or modified executable is refused without mutation
- [ ] Automated checks cover patch identity, idempotency, rollback, configuration, and aspect-ratio calculations
- [ ] The repository contains no original executable or proprietary game assets

## Plan
Use the W-0035 dossier to locate the narrowest rendering seam, prove the geometry policy with a small fixture, and then package the minimal reversible compatibility mechanism with target-hash checks.
