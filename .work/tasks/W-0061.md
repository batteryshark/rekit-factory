## Goal
Make the durable investigation JSON export a canonical outcome consumer without allowing model-authored report prose to assert validation, resolution, or operator acceptance.

## Requirements
- [x] Embed the complete versioned canonical outcome projection and semantic identity
- [x] Render the export after the terminal run transition and event are committed
- [x] Give every exported worker report its canonical identity, facets, and diagnostics
- [x] Preserve model prose only as an explicitly non-authoritative worker note
- [x] Remove the ambiguous exported worker status field
- [x] Capture all SQLite-backed projection inputs under one read transaction

## Acceptance Criteria
- [x] A completed export projection exactly matches the canonical completed run snapshot
- [x] A worker note claiming “validated, solved, and accepted” changes no canonical facet
- [x] Consumers can distinguish report publication from validation and acceptance
- [x] The export contains projection schema, vocabulary, canonical bytes, and semantic SHA-256
