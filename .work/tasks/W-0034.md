## Goal
Select model, reasoning effort, latency mode, context budget, and retry behavior by evidence-backed research phase instead of using one expensive/default profile for every worker turn.

## Requirements
- [ ] Extend public model-profile metadata with supported reasoning/effort controls, latency tier, context/output limits, structured/tool behavior, cache behavior, concurrency, retry policy, and configured cost units without exposing credentials
- [ ] Define routing policy by work class such as triage, extraction, hypothesis generation, discriminating test design, validation, synthesis, and operator-facing explanation
- [ ] Permit an initial operator-selected fixed policy plus an adaptive policy whose escalation/de-escalation rules are deterministic and inspectable
- [ ] Require evidence such as validation failure, unresolved ambiguity, or benchmarked quality need before escalating to a higher-cost/effort profile
- [ ] Bound per-call, worker, epoch, campaign, and provider concurrency/cost; fallback cannot silently weaken proof or structured-output requirements
- [ ] Record requested/effective profile, routing reason, capability compatibility, retries/fallbacks, usage, cache results, latency, and outcome
- [ ] Evaluate routing policies through ReverseBench repeated trials and compare quality, false positives, reproduction, time, intervention, and cost
- [ ] Pin routing policy/version to each run so later policy changes do not rewrite historical interpretation
- [ ] Keep operator overrides explicit and durable; workers cannot select an unregistered provider or exceed campaign ceilings
- [ ] Handle unavailable/rate-limited providers with bounded backoff and compatible fallback or a reasoned blocked state

## Acceptance Criteria
- [ ] A mixed fixture uses a low-cost profile for bounded recon and a higher-assurance profile only for the configured judgment/validation phase
- [ ] Adaptive escalation occurs for a defined evidence-backed condition and does not occur for mere worker preference
- [ ] An incompatible fallback is rejected rather than silently dropping required tool calling, schema, continuation, or reasoning semantics
- [ ] Fixed and adaptive policies produce comparable ReverseBench reports with complete profile/version/cost attribution
- [ ] Provider outage/rate-limit tests remain within retry/time/cost ceilings and preserve resumable state
- [ ] Credentials never enter routing records, events, reports, or browser payloads

## Plan
Finish W-0013's profile contract, define pure routing decisions and records, integrate by work purpose, then use W-0028 to establish whether adaptive policies materially improve quality/cost before making them defaults.
