## Goal
Reconcile every exact Parallels lifecycle plan against bounded before/after inventory without invoking a provider command.

## Requirements
- [x] Model bounded immutable VM, snapshot, state, source, and observation-chain inventories
- [x] Decide execute, already-applied, conflict, or unknown for clone, start, stop, snapshot creation/switch, and delete
- [x] Discover provider VM and snapshot UUIDs only from an exact observed completed state
- [x] Emit the canonical success envelope accepted by the durable effect parser only for already-applied plans
- [x] Treat an unchanged post-effect observation as unknown so recovery cannot blindly rerun the effect
- [x] Reject adapter mismatch, broken observation chains, boolean versions, unhashable states/decisions, unknown fields, duplicate UUIDs, and oversized inventories

## Acceptance Criteria
- [x] Every W-0081 plan kind has execute and completed-state coverage
- [x] Clone/snapshot naming collisions and missing provider identities fail closed
- [x] Crash recovery can derive a canonical success capture from observed provider state or stop at conflict/unknown
- [x] The module contains no subprocess or inventory-collection capability
