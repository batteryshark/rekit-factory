## Goal
Define and implement a safe, transport-neutral remote-worker protocol for platform-restricted tools without making any hypervisor, operating system, or machine-specific adapter a Factory dependency.

## Requirements
- [x] Local and remote tool calls share one invocation and result envelope
- [x] Worker leases expose setup, reset, teardown, cancellation, attachment, dirty-state, and explicit network/mount policy contracts without claiming a particular adapter's disposal mechanism
- [x] Artifacts and logs return to the Muster run with provenance
- [x] Secrets and host mounts are opt-in and least-privilege
- [x] The production controller selects a capability-compatible remote worker and constructs its invocation from the immutable W-0022 scope revision, exact action/endpoint/account/credential intent, and staged target hash
- [x] Remote invocation state, lifecycle authority, events, results, tombstones, and artifact transfer survive restart without duplicate execution or silent local fallback

## Acceptance Criteria
- [x] An architecture decision compares VM and container options by platform, isolation, automation, and observability
- [x] Deterministic local and authenticated HTTP adapters exercise capability routing, restart, cancellation, cleanup, artifact verification, and fail-closed provenance through the same protocol
- [x] The generic protocol does not claim disposable-image or native-Windows guarantees that only a real adapter can prove
