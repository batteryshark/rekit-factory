## Goal
Define a safe worker protocol for platform-restricted tools, beginning with a disposable Windows VM capable of x64dbg-class workflows.

## Requirements
- [ ] Local and remote tool calls share one invocation and result envelope
- [ ] Images are disposable, snapshots are resettable, and network policy is explicit
- [ ] Artifacts and logs return to the Muster run with provenance
- [ ] Secrets and host mounts are opt-in and least-privilege

## Acceptance Criteria
- [ ] An architecture decision compares VM and container options by platform, isolation, automation, and observability
- [ ] A minimal Windows proof of concept runs one benign fixture and returns logs and artifacts
