## Goal
Give `factory-outcomes/v1` a reproducible semantic content identity so consumers can compare complete outcome meaning without mistaking partial source watermarks for projection identity.

## Requirements
- [ ] Define one versioned canonical byte domain over every public semantic field in the projection, excluding only explicitly non-semantic observation metadata such as independent source watermarks
- [ ] Derive a lowercase SHA-256 identity with no timestamps, randomness, process state, locale, or insertion-order dependence
- [ ] Make the identity recomputable from the returned public projection after removing the identity field itself
- [ ] Keep source watermarks visible as diagnostics while proving that watermark movement alone neither changes nor masquerades as semantic identity
- [ ] Change identity whenever an entity, facet, authority, parent, diagnostic, degraded flag, schema version, or vocabulary meaning changes
- [ ] Document that this is an outcome-projection content identity, not an atomic revision of every ledger table, project-memory source, run snapshot field, or proof byte

## Acceptance Criteria
- [ ] Reordering workers, work items, dossiers, decisions, or mapping insertion order produces byte-identical semantic payloads and the same identity
- [ ] Rebuilding unchanged canonical inputs produces the same identity across process restarts
- [ ] A meaningful raw/normalized state, parent relation, publication, acceptance, or diagnostic change produces a different identity
- [ ] Changing only the independently observed event-rowid or memory-sequence watermark leaves semantic identity unchanged
- [ ] A standalone verifier can recompute and validate the identity from the public projection without private state

## Plan
Specify the exact canonical domain first, add a small pure verifier/helper, bind it at the end of the existing deterministic full fold, and cover adversarial ordering plus semantic/non-semantic changes before any UI or SSE consumer adopts it.

## Notes
This is an enabling slice for full/incremental parity and consumer convergence. It does not claim a cross-store transaction, replace the full-fold projection, or make source watermarks safe as ETags.
