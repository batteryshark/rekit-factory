## Goal
Turn strict outbox, preference, and transport contracts into a restart-safe sender without making delivery authoritative over Factory state.

## Requirements
- [x] Persist exact preference identities, schedules, channel references, and delivery dispositions outside run-authored payloads
- [x] Supervise due immediate/batch/digest/escalation work with bounded leases, retry, and external idempotency
- [x] Provide an environment-owned local desktop implementation and an opt-in HTTPS webhook implementation with SSRF/redirect/timeout limits
- [x] Resolve opaque credential references only at send time and never store endpoints, tokens, or hostile transport prose in run/outbox payloads
- [x] Supersede self-resolved alerts and preserve failed delivery as non-blocking Mission Control state

## Acceptance Criteria
- [x] Restart at every schedule/send boundary converges to one provider-visible idempotency key
- [x] Unavailable and ambiguous receivers exhaust bounded retry without blocking or losing the canonical decision
- [x] Batch, digest, quiet-hours, and escalation schedules survive restart under a fake clock
- [x] Redirects, private-network endpoints, credential leaks, and cross-channel requests fail closed

Concrete adapters remain bounded to 30 seconds; custom injected adapters must honor that protocol contract.

## Completion Summary
Notification delivery is now a durable, lossy side system: it can schedule, reconcile, lease, retry, send, fail, and supersede without becoming authoritative over canonical Factory progress. The live worker is bounded, fair, restart-safe, and channel-selection aware.
