## Goal
Turn vague vulnerability/research leads into explicit, testable, evidence-linked hypotheses that Factory can schedule, disprove, refine, compare, and resume.

## Requirements
- [x] Define a stable hypothesis record with claim, scope, expected observation, falsifier, proposed test, confidence, provenance, owner workstream, stop condition, and status
- [x] Use an explicit lifecycle such as proposed, queued, testing, supported, contradicted, disproved, reproduced, retired, and blocked; document allowed transitions
- [x] Separate observations from interpretations and require evidence references for confidence or status changes
- [x] Allow multiple competing hypotheses without collapsing uncertainty into one model narrative
- [x] Translate approved hypothesis tests into deterministic Muster work with stable dedupe keys, dependencies, costs, and authorization requirements
- [x] Let workers propose hypotheses and tests but not decide overall run completion or silently broaden target scope
- [x] Record every attempt, including negative outcomes and environmental ambiguity, and feed relevant results into future scheduling
- [x] Prioritize tests by information gain, risk, cost, prerequisites, and likelihood of discriminating between competing explanations
- [x] Detect semantic duplicates and refinements without merging distinct scopes or erasing provenance

## Acceptance Criteria
- [x] A fixture produces two competing hypotheses; one is disproved and one supported by different evidence-backed tests
- [x] Restarting between proposal and test completion leases each hypothesis test exactly once and retains its history
- [x] A worker cannot mark a hypothesis supported without cited observations, nor schedule an out-of-scope test
- [x] A disproved hypothesis prevents equivalent follow-up work while permitting a materially revised claim with new evidence
- [x] Exhausted stop conditions close or block the hypothesis with an explicit reason instead of generating an endless retry loop
- [x] Hypothesis state is inspectable independently from raw model messages

## Plan
Implement pure schema and transition validation, then durable scheduler integration, then model action contracts and prioritization. Mission Control presentation belongs to W-0033.

## Notes
This extends evidence-driven FollowUpProposal work rather than creating a second scheduler. A hypothesis is a research entity; its executable tests remain ordinary Muster work items.
