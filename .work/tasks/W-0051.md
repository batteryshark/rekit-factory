## Goal
Define strict versioned campaign, epoch, budget, policy, checkpoint, progress-signal, and terminal-outcome contracts before any unattended controller loop is allowed to run.

## Requirements
- [ ] Bind campaign identity to project/goal, immutable scope revision, operator policy, completion criteria, cumulative ceilings, and exact component versions
- [ ] Define finite epoch identity, proposed work, per-epoch ceilings, start/end checkpoints, progress observations, next actions, and terminal reason
- [ ] Represent work/concurrency/retry/token/cost/time/tool/network/artifact ceilings with exact units and hard-versus-soft semantics
- [ ] Define requested/running/waiting/suspended/completed/exhausted/blocked/stopped/policy-stopped/failed transitions with owners and allowed predecessors
- [ ] Keep scope or hard-ceiling increases representable only as exact-content operator decisions
- [ ] Canonicalize every contract deterministically and reject unknown fields, non-finite/unbounded values, inconsistent totals, and ambiguous terminal states

## Acceptance Criteria
- [ ] Canonical identity changes on every scope, goal, policy, budget, completion, or component-version change
- [ ] No contract can express an infinite epoch, undefined cumulative spend, implicit scope expansion, or success without a reasoned terminal outcome
- [ ] Unknown future states can be preserved by W-0026 projections without being treated as successful
- [ ] Round-trip, malformed, overflow, unit-confusion, and impossible-transition fixtures fail deterministically

## Notes
This is contracts only. Persistence, progress policy, scheduling, and UI remain W-0052 through W-0055.
