## Goal
Let an operator select whether finding notifications begin at reproduced or accepted while preserving one exact proof-linked delivery.

## Requirements
- [x] Add a server-declared finding-stage policy with `reproduced` and `accepted` values
- [x] Bind configuration revisions and candidate identity to the selected stage
- [x] Suppress reproduced delivery when accepted-only is selected without losing later acceptance
- [x] Preserve campaign-owned dossier qualification, redaction, replay, and supersession behavior
- [x] Expose the bounded selection in Mission Control without accepting notification prose or payloads

## Acceptance Criteria
- [x] One finding crossing both stages emits exactly one notification at the configured stage
- [x] Changing policy cannot replay an already delivered equivalent or strand an actionable transition
- [x] The delivered deep link resolves to the exact qualified proof dossier when one exists
- [x] Reconnect, restart, stale revision, and hostile browser fields fail closed
