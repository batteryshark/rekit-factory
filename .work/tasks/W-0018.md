## Goal
Turn durable investigation findings into reviewable re-kb contribution candidates without treating raw logs, model output, or unverified theories as publishable knowledge.

## Requirements
- [ ] Candidate extraction separates durable reusable knowledge from target-specific evidence, behavioral instructions, code, credentials, and unsafe material
- [ ] Deduplication searches existing concepts before proposing creation or enrichment
- [ ] Every proposal distinguishes observation from interpretation and carries run/artifact/event provenance
- [ ] Bundle writes require an explicit operator decision and use a staging branch/worktree rather than modifying re-kb during an investigation
- [ ] Accepted changes update the concept, parent index, newest-first log, and pass strict OKF validation

## Acceptance Criteria
- [ ] A fixture run produces a reviewable proposal but no knowledge-base mutation before approval
- [ ] Approval creates a conformant staged change with citations; denial leaves the bundle untouched
- [ ] Secrets, raw binaries, personal data, persistence payloads, and weaponized chains are rejected before staging

## Plan
Specify the candidate/provenance schema and review lifecycle first. Integrate with re-kb `INGEST.md` and strict validation only after read-only retrieval is stable.

## Notes
Logging and knowledge publication are different lifecycles. Logs remain append-only run evidence; only distilled, deduplicated, reviewed concepts enter re-kb.
