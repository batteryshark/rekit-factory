## Goal
Implement the smallest clean-room `xio.dll` compatibility layer that satisfies Frenzy Express keyboard input needs against the reviewed W-0037 contract.

## Requirements
- [ ] Implement only the documented exports, calling conventions, state transitions, and keyboard behavior required by the target
- [ ] Keep platform input collection behind a narrow adapter so contract behavior can be tested deterministically without physical key injection
- [ ] Match documented initialization, polling/transitions, focus handling, repeat behavior, error returns, and shutdown semantics
- [ ] Fail safely and observably for unsupported devices or unimplemented calls rather than pretending to emulate the full original library
- [ ] Embed contract version and supported target hashes in diagnostics while keeping browser-facing paths and identifiers redacted
- [ ] Distribute only independently written compatibility code, tests, and documentation

## Acceptance Criteria
- [ ] W-0037 conformance vectors pass without loading the original `xio.dll`
- [ ] The authorized baseline accepts keyboard input for its documented critical path with the compatibility library substituted
- [ ] Simultaneous keys, transitions, focus loss/recovery, invalid call order, and clean shutdown behave as specified
- [ ] Unsupported functionality is deterministic and documented
- [ ] A clean build and test require no proprietary files; optional live validation consumes an operator-supplied target outside the repository

## Plan
Freeze a reviewed W-0037 contract version, build a pure state-machine core and fake input adapter, then add the narrow platform adapter and separately authorized target validation.

## Notes
This task must not begin from decompiled implementation text. Its input is the behavioral contract and conformance corpus produced by W-0037.
