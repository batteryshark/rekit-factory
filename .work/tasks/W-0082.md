## Goal
Create a durable, fail-closed effect/result boundary around W-0081's exact Parallels plans without invoking a real command or changing VM state.

## Requirements
- [x] Persist the complete immutable command plan before calling an injected exact-argv runner
- [x] Bind operation, plan, adapter, lifecycle target, argv, capture, and result identities
- [x] Reject operation-ID reuse, changed argv, unknown fields, duplicate JSON keys, malformed UUIDs, and unbound runner observations
- [x] Converge a crash after the provider boundary through an explicitly idempotent operation/plan runner contract
- [x] Atomically fsync the intent/result journal and replay completed success or failure without another runner call
- [x] Serialize separate writers without lost updates and fail closed on unsafe parent, database, or sidecar replacement
- [x] Bound journal, sidecar, record, capture, and operation-count growth and expose an idempotent close lifecycle
- [x] Keep subprocess execution and real VM mutation outside this dependency-safe slice

## Acceptance Criteria
- [x] Focused tests cover intent persistence, completed replay, post-effect crash convergence, failed effects, conflict rejection, and corrupt journals
- [x] Raw provider output is bounded and only strict one-line JSON success/error envelopes are accepted
- [x] The durable result contains no credentials, host paths, or unbounded provider output
- [x] Adversarial tests cover symlink, inode, parent, sidecar, closed-store, oversized-store, and concurrent-writer behavior
