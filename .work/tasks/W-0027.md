## Goal
Allow Factory to work unattended for hours or days through bounded research epochs, durable checkpoints, and evidence-based replanning without implementing an infinite prompt loop.

## Requirements
- [x] Define campaign identity, goal, immutable authorization scope revision, campaign status, current epoch, cumulative budgets, completion criteria, and operator policy
- [x] Divide execution into finite epochs with explicit work/concurrency/retry/token/cost/time/tool/network/artifact ceilings
- [x] Checkpoint canonical state at epoch boundaries and resume exactly once after process, host, model, or worker interruption
- [x] Evaluate progress using new material evidence, hypothesis resolution, coverage movement, reproduced findings, and capability-gap changes rather than model claims of activity
- [x] Detect no-progress, repeated equivalent attempts, validation churn, dependency deadlock, flapping environments, and notification spam
- [x] Back off, reprioritize, suspend, or terminate according to configurable thresholds; never spend indefinitely to prove liveness
- [x] Require operator approval to broaden target/action scope, raise hard cumulative ceilings, use forbidden capabilities, or continue after defined risk thresholds
- [ ] Support serial and parallel workstreams while preserving global budget and exact leasing/dedupe invariants
- [x] Distinguish successful completion, exhausted search, blocked/inconclusive completion, operator stop, policy stop, and infrastructure failure
- [ ] Produce a bounded handoff/checkpoint summary after each epoch with evidence references and the next proposed actions

## Acceptance Criteria
- [ ] A simulated overnight campaign survives repeated controller restarts and completes each work item/epoch transition exactly once
- [ ] A no-novelty fixture terminates or asks for direction after the configured threshold without exhausting an unrelated global budget
- [ ] A productive fixture adapts from recon to hypothesis tests to validation while staying within every cumulative ceiling
- [ ] Concurrent campaigns cannot borrow one another's scope, budget, workers, evidence, or approvals
- [ ] Raising a hard ceiling or changing scope requires a durable exact-content operator decision
- [ ] Final status explains what was achieved, what remains unknown, why work stopped, and which evidence supports that account

## Plan
Implement the campaign/epoch state machine and accounting, then progress scoring/stuck detection, then adaptive scheduling and operator controls. Prove with deterministic fake clocks/crash points before any real unattended run.

## Notes
The useful behavior is persistent autonomous progress. Infinite workers, undefined completion, and subscription-limit consumption are explicitly rejected. A campaign may have open-ended discovery goals only if every epoch and cumulative resource/risk boundary remains finite.
