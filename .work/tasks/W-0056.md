## Goal
Prove that a process/database failure while exposing a sealed proof dossier cannot make a partial publication visible and that an exact retry converges without changing the content-addressed files or duplicating ledger state.

## Requirements
- [x] Inject a deterministic failure during the artifact insertion sequence and prove the complete dossier artifact set rolls back
- [x] Inject a deterministic failure at the final `dossier.published` event and prove all preceding artifact inserts roll back with it
- [x] Preserve the already materialized content-addressed directory as invisible staging state after the failed transaction
- [x] Retry from that exact directory and publish five bound artifacts plus one publication event exactly once
- [x] Prove every sealed file remains byte-identical across failure and recovery

## Acceptance Criteria
- [x] Neither injected boundary leaves any `proof-dossier` artifact or `dossier.published` event visible
- [x] Recovery produces the original manifest identity, five artifacts, and one event
- [x] A repeated publication remains idempotent under the existing deterministic export test
- [x] The focused crash suite passes without adding production-only fault controls

## Plan
Use SQLite abort triggers in the integration test so the real production transaction and publisher execute unchanged. Retain the sealed directory between the injected abort and retry to exercise the actual orphan-materialization recovery path.

## Notes
This slice proves the SQLite dossier publication boundary. It does not claim cross-store atomicity with project-memory JSONL, crash coverage for every unrelated Factory event, or completion of W-0026's consumer/SSE migration work.
