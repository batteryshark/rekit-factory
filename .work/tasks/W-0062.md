## Goal
Add a small provider-neutral supervisor that durably converges expired, failed, and explicitly cancelled range leases toward cleanup without claiming real infrastructure behavior.

## Requirements
- [x] Persist a pending expiration or destruction operation before calling the adapter
- [x] Acknowledge an adapter result only after it can be durably recorded
- [x] Reuse the exact pending operation ID after an ambiguous controller crash
- [x] Allocate a distinct attempt ID only after a retryable failure has been durably recorded
- [x] Record bounded cleanup audit entries, last failure, blocked state, and terminal convergence per range
- [x] Store supervisor state in strict, bounded, atomic POSIX JSON without following root or state-file symlinks
- [x] Keep provisioning, isolation, provider credentials, provider durability, and real cleanup effects outside this slice

## Acceptance Criteria
- [x] An overdue deterministic lease is expired and then destroyed as two separately durable effects
- [x] Failed and explicitly cancelled deterministic leases can be destroyed before their expiry time
- [x] A retryable injected cleanup failure converges through a distinct subsequent attempt
- [x] Restart after adapter success but before supervisor acknowledgement replays one exact operation without duplicating the adapter transition
- [x] Duplicate-key, malformed UTF-8, oversized, symlinked, and failed-write state cases fail closed or retain the last good checkpoint
- [x] Focused supervisor and range tests pass

## Plan
Keep the supervisor as an adapter-facing module independent of the production controller and HTTP API. Bracket each effect with atomic state writes and exercise restart ambiguity against the existing deterministic adapter checkpoint.

## Notes
This is controller-side reconciliation contract work only. The adapter remains responsible for independently durable idempotency and for proving any real provisioning, network isolation, reset, destruction, or disposal property.
