**Target repo:** rekit

## Context

- Epic **E3**. Plan: `parallax/docs/rekit-epic.md` §E3. Depends on E0 (T-047).
- Skills become universal `SKILL.md` + `scripts/` folders under `REKIT_HOME`, discovered by
  filesystem convention (no pip/entry points).
- **Tree-sitter is tooling, not taxonomy** — a code-understanding *skill* runs the parser and emits
  atoms/signatures conforming to the parallax schema. Parsers live here, never in parallax.

## Goal

Filesystem skill discovery + shared `~/.rekit/bin` + per-skill uv venv + a **searchable registry**
(`find_skills(intent)`), and relocate existing tools (asar/archive/decompilers) into skill folders.

## Acceptance Criteria

- [ ] Dropping a `SKILL.md`+`scripts/` dir makes it discoverable with zero install (incl. one
      authored in another project).
- [ ] Decompile skills run from `REKIT_HOME` against shared `bin/` (ghidra/jadx) via host-gating.
- [ ] Each skill carries its own uv venv; heavy deps (tree-sitter, etc.) stay isolated per skill.
- [ ] `find_skills(intent)` surfaces a skill by description/keywords — e.g. the creds-in-a-text-file
      case with no special artifact kind.
- [ ] A code-understanding skill parses via tree-sitter and emits **parallax-schema** atoms.

## Notes

- Existing host-gating already takes `env=`/`paths=`, so shared-bin resolution is a small change.
