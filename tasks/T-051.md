## Context

- Epic **E4**. Plan: `parallax/docs/rekit-epic.md` §E4. Depends on E3 (T-050).
- Don't expose every skill to every goalpack. rekit is the exposure authority — the harness only
  ever sees the scoped per-turn set.

## Goal

A scoping resolver — `(kinds present in ledger) ∩ (capabilities the goalpack requested)`, filtered
by **trust tier** — plus the human channel (`ask_human` / `present_choices`).

## Acceptance Criteria

- [ ] A read-only goalpack never sees a destructive/network skill.
- [ ] Skills carry a trust tier (read-only / network / executes-untrusted / destructive) in
      `SKILL.md`; tiers drive auto-run vs. gated escalation per goalpack policy.
- [ ] "Allow network?" renders as a choice and blocks the loop until answered (pairs with sandbox).
- [ ] **Proactive consultation** works: a direction fork ("go deeper or move on?") reaches the user
      mid-loop, not only hard gates.

## Notes

- The human channel's active half is shared with the UI (E7.3) and the network gate (sandbox work).
