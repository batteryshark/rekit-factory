**Target repo:** rekit-factory

## Context

The persistent ledger already records artifacts, findings, derivations, and
missing-capability leads. The run log records liveness, and the inbox records
questions that require a human. Long-running investigations still lose the
reasoning layer across runs: goal pivots, workstreams, failed attempts, durable
decisions, research questions, theories, and prioritized next actions.

The design of record is [`docs/project-memory.md`](../docs/project-memory.md).

## Goal

Add an event-sourced project-memory protocol that the loop, harness context, and
Mission Control can consume, with deterministic Markdown as an export rather
than a second writable source of truth.

## Acceptance Criteria

- [ ] `memory.jsonl` uses the standard event envelope and folds into a stable
      `ProjectMemory` snapshot.
- [ ] Goals/pivots, workstreams, attempts, decisions, research questions,
      theories, and next actions have typed append APIs and replay tests.
- [ ] Updates use stable entity IDs and retain resolved/rejected history.
- [ ] Evidence is referenced through ledger/run sequences, artifact hashes,
      paths, or URIs instead of duplicating bulky output.
- [ ] Harness context receives a bounded active-memory projection.
- [ ] A deterministic `tracker-markdown` export covers the legacy tracker views.
- [ ] Unknown event types remain forward-compatible no-ops.
- [ ] Mission Control can expose the folded memory without becoming a source of
      truth itself.

## Notes

- Implement in delivery slices; protocol/fold comes before UI.
- Keep `needsYou` derived exclusively from unresolved human-inbox questions.
- A legacy Markdown importer is optional until there is a real migration case.
